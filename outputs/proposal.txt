Introduction
CosmereLang reimagines common programming constructs—such as data types, control
flow, and function semantics—using meaningful, lore-inspired terminology (e.g., Metal,
Essence, Ascend, ShardForm) while preserving the logical foundations of imperative
programming. The language is designed to remain simple, deterministic, and
compiler-friendly, making it suitable for implementation using Lex/Flex for lexical analysis
and Yacc/Bison for syntax analysis.

The primary motivation of this project is educational: to demonstrate how a high-level
language can be designed, tokenized, parsed, and semantically validated, while also
showing how creative inspiration can coexist with formal compiler construction principles.
CosmereLang serves as both a functional programming language prototype and a practical
exploration of compiler design concepts taught in the course.

Unique Features

Safe Null-Aware Operator (??)

CosmereLang includes a null-aware operator (??) that safely handles missing or
undefined values by returning a fallback expression when the primary value is null. This
reduces the need for explicit conditional checks and helps prevent runtime errors caused
by invalid input. Unlike C, which lacks built-in null safety, this feature promotes safer and
more readable code while remaining simple to implement at the compiler level.


Deferred Block Execution (defer)

CosmereLang provides a deferred execution mechanism that allows a block of code to be
executed automatically when the surrounding scope exits. This ensures that cleanup or
final actions are performed reliably without manual control flow management. Since C has
no native support for deferred execution, this feature improves code structure and
maintainability while being straightforward to implement using scope tracking in the
compiler.
